# 臺灣營業稅申報模組

本模組適用於 Odoo 19 Community，用來維護臺灣營業稅申報欄位、產生營業稅媒體申報檔（ZIP），並匯出內建官方空白申報書的 Excel。

## 支援環境

- Windows 自架版 Odoo 19 Community
- Linux 自架版 Odoo 19 Community
- Odoo.sh
- 不支援 Odoo Online

## 安裝前提

- 已安裝 `account` 模組
- 不需要額外安裝第三方 Python 套件
- 模組資料夾已放入 Odoo 可讀取的 `addons` 路徑，或已部署到 Odoo.sh 分支
- 操作者具有應用程式安裝與升級權限
- 建議使用者具有會計管理權限

## Windows 與 Linux 的 UI 安裝方式

只要模組程式碼已放入正確的 `addons` 路徑，之後可由 Odoo 管理員透過 UI 完成安裝。

1. 以管理員登入 Odoo。
2. 進入 `應用程式`。
3. 如果模組尚未出現，先啟動開發人員模式，再到 `應用程式` → `更新應用程式列表`，按 `更新`。
4. 搜尋 `臺灣營業稅申報模組`。
5. 在模組卡片上按 `啟動`。
6. 安裝完成後，到公司資料填寫 `稅籍編號(9碼)` 與申報人相關資料。
7. 確認公司資料、單據畫面與匯出精靈均已出現。

## Windows 與 Linux 的 UI 升級方式

1. 先備份資料庫。
2. 將新版模組覆蓋到原本相同的 `addons` 路徑。
3. 如果本次更新包含 `.py` 檔案變更，先重啟 Odoo 服務，再進入 Odoo。
4. 以管理員登入 Odoo，進入 `應用程式`。
5. 搜尋 `臺灣營業稅申報模組`。
6. 在模組卡片右上角按垂直省略號，然後按 `升級`。
7. 升級完成後，依下方驗收清單確認畫面與功能。

## Odoo.sh 的 UI 安裝與升級方式

1. 將模組提交到 Odoo.sh 使用的 Git 分支。
2. 等待 Odoo.sh 完成建置。
3. 以管理員登入對應環境的 Odoo。
4. 進入 `應用程式`。
5. 如果模組尚未出現，先啟動開發人員模式，再到 `應用程式` → `更新應用程式列表`，按 `更新`。
6. 搜尋 `臺灣營業稅申報模組`。
7. 依需要按 `啟動` 或 `升級`。

## 安裝與升級後驗收清單

- 公司資料中可看到 `臺灣營業稅申報` 分頁
- 發票或帳單畫面中可看到 `臺灣營業稅申報` 分頁
- 可看到 `自動帶入發票字軌與號碼`
- 可看到 `自動帶入課稅別與扣抵`
- 可從功能表開啟 `臺灣營業稅申報`
- 匯出精靈可正常開啟
- 可使用 `產生營業稅媒體申報檔 (ZIP)`
- 可使用 `產生進銷項核對表 (Excel)`
- 舊資料可正常開啟與儲存

## 升級相容性說明

- 依賴模組只有 `account`
- 不需要額外 Python 套件，Windows 與 Linux 自架 Odoo 可直接載入
- 保留既有相容欄位，降低舊資料升級風險
- 自動帶入 trigger 欄位為非儲存欄位，不需要人工 migration script
- 沒有作業系統相依程式碼

## 不屬於本模組可避免的安裝或升級問題

- 模組資料夾未放入 `addons` 路徑
- Odoo 服務沒有讀到新的 `addons` 路徑
- 更新了 `.py` 檔案但未先重啟 Odoo
- 使用者沒有應用程式安裝或升級權限
- Odoo Online 環境
- 其他第三方模組修改同一個畫面而產生衝突

## 發票字軌與號碼自動帶入說明

`自動帶入發票字軌與號碼` 目前是預留整合機制。

- 本模組本身不直接提供發票來源欄位
- 未來可由其他模組覆寫 hook 提供字軌與號碼來源
- 如果尚未整合來源模組，勾選後只會提示目前無可用整合欄位

## 發行建議

- GitHub 發行時不要包含 `__pycache__`、`.pyc`、`dist` 或暫存檔
- 升級正式環境前，先在測試資料庫按一次 `升級`
- 正式安裝或升級後，再讓會計使用者開始匯出申報檔
